1. About Kaggle

1.1 What is Kaggle?

Kaggle is an online platform for data science and machine learning practitioners. It originally started as a hub for data science competitions. Over time, it's grown into a broader ecosystem with support for running Jupyter Notebooks, hosting datasets, sharing models, and even offering short courses via guided notebooks.

1.2 How I use it to code (comparison with Colab)

Kaggle also allows you to use their platform to code, very similar to Google Colab. But there are a few reasons I prefer Kaggle over
  • Free GPU/TPU Access
    While both Kaggle and Colab offer free accelerator access, Kaggle explicitly displays a weekly quota (e.g., 30 hours of GPU time), so you can plan around that.
  • Cloud Execution & Saving
    Kaggle's "Save & Run All (Commit)" feature executes your entire notebook on a separate cloud instance. That means - you can close your browser, and it will continue running (up to a time limit of 9 hours). And outputs and generated files are saved with the version!
  • Versioning
    Every time you "Commit", Kaggle saves a snapshot of your notebook and its outputs on their website. You can easily share, review and revert to previous versions!
  • GitHub Integration
    You can also link your GitHub, so that Kaggle automatically pushes your notebook to GitHub at the end of each commit, allowing you to have another backup of everything.
  • Faster Dataset Access
    You can use datasets hosted directly on Kaggle (and even upload your own private datasets), which is usually significantly faster than accessing Google Drive files on Colab.
  • Scheduled Runs
    Kaggle allows you to schedule notebook runs (daily, weekly, etc.), which can be very useful for automated tasks like data updates or model retraining, especially when you're working with live data.

2. Kaggle Competitions

Kaggle is most known for its competitions, where individuals and teams compete to build the best models and submit solutions for a given challenge. Anyone can host a competitions, and there are often big monetary prizes on offer for the best solution.

2.1 In-class competitions

Kaggle allows hosting in-class competitions (like for PHYS 664), great for learning the platform and its leaderboard system designed to prevent overfitting:

  • Public Leaderboard: Score on a subset of test data, visible during the competition.
  • Private Leaderboard: Final score on the remaining test data, revealed after the competition ends.
This encourages models that generalize well.

2.2 Real-world competitions

Exploring public competitions is a great way to practice skills on real-world problems! If you can win, there's large monetary rewards available. But more importantly, most winners generally share their winning code, making it a great resource to learn!
